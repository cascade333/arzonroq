# Arzonroq.uz — инструкция по запуску

## 0. Клонирование на VPS

```bash
cd /var/www   # или другая папка, где лежит Yaqin
git clone <URL_твоего_репозитория> arzonroq
cd arzonroq
```

## 1. Установка зависимостей

```bash
npm install
```

## 1.5. PostgreSQL (если делишь сервер с другим проектом)

Если Postgres уже установлен (например, для Yaqin), просто создаём новую
базу и пользователя — существующие базы не трогаются:

```bash
sudo -u postgres psql
```

В консоли psql:

```sql
CREATE DATABASE arzonroq;
CREATE USER arzonroq_user WITH ENCRYPTED PASSWORD 'придумай_надёжный_пароль';
GRANT ALL PRIVILEGES ON DATABASE arzonroq TO arzonroq_user;
ALTER DATABASE arzonroq OWNER TO arzonroq_user;
\q
```

Если Postgres не установлен вообще:

```bash
sudo apt update
sudo apt install postgresql postgresql-contrib -y
sudo systemctl enable --now postgresql
```

Затем повтори команды создания базы выше.

## 2. Переменные окружения

Скопируй `.env.example` в `.env` и заполни реальными значениями:

```bash
cp .env.example .env
nano .env
```

- `DATABASE_URL` — например: `postgresql://arzonroq_user:придумай_надёжный_пароль@localhost:5432/arzonroq?schema=public`
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `ADMIN_SESSION_SECRET` — сгенерировать: `openssl rand -base64 32`
- `NEXT_PUBLIC_BASE_URL` — домен сайта, например `https://arzonroq.uz`
- `MANAGER_TELEGRAM_CHAT_ID` — chat_id менеджера, которому приходят заявки на бронирование
- `CRON_SECRET` — любая случайная строка, защищает cron-эндпоинты от посторонних вызовов
- `UPLOADS_BASE_URL` — см. раздел "⚠️ Загруженные фото после сборки" ниже

## ⚠️ Загруженные фото после сборки (`/public/uploads`)

Next.js в продакшен-режиме обрабатывает папку `public/` как снимок,
сделанный во время `npm run build` — файлы, появившиеся в ней **после**
сборки (то есть все реальные пользовательские загрузки фото техпаспорта
и авто через `/sell`), физически лежат на диске, но сам Next.js отдаёт
на них 404. Это особенность платформы, не баг конкретно нашего кода.

Решение — раздавать `/uploads/` через **Nginx напрямую**, в обход
Next.js, прямым доступом к файлам на диске:

```nginx
location /uploads/ {
    alias /var/www/arzonroq/public/uploads/;
    access_log off;
    expires 30d;
}
```

Это нужно добавить в тот же server-блок Nginx, что проксирует основной
трафик на Next.js (порт 3001) — после настройки домена с единым портом
(443/80) пути к фото в коде остаются относительными (`/uploads/...`) и
работают сразу, без `UPLOADS_BASE_URL`.

**Временный случай** (домен ещё не подключен, sayt доступен только по
`http://IP:порт`, и Nginx из-за конфликта с другим проектом на сервере
не может слушать порт 80/443 под этим же IP): раздайте `/uploads/` через
отдельный порт Nginx (например 8080), и заполните `UPLOADS_BASE_URL` в
`.env` явным адресом этого порта:

```
UPLOADS_BASE_URL="http://5.129.255.189:8080"
```

После настройки нормального домена не забудьте **очистить**
`UPLOADS_BASE_URL` (оставить пустым), пересобрать и перезапустить —
иначе фото продолжат запрашиваться по старому временному адресу.

## 6. Сборка и запуск

```bash
npm run build
PORT=3001 pm2 start npm --name arzonroq -- start -- -p 3001
pm2 save
```

Проверить, что процесс поднялся и не конфликтует с Yaqin:

```bash
pm2 list
pm2 logs arzonroq
```

Nginx — проксировать на порт 3000 (по умолчанию у Next.js). Так как на
сервере уже работает Yaqin, выбери порт, который не занят (например, 3001),
и пропиши его в `.env` как `PORT=3001` или через `pm2`/`next start -p 3001`.

Пример конфига `/etc/nginx/sites-available/eng-arzon.uz`:

```nginx
server {
    listen 80;
    server_name eng-arzon.uz www.eng-arzon.uz;

    client_max_body_size 15M;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        # КРИТИЧНО: без этих двух заголовков rate-limiting (см. lib/rateLimit.ts)
        # не увидит реальный IP посетителя и будет считать всех одним клиентом.
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/eng-arzon.uz /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

Для HTTPS — Certbot как обычно: `sudo certbot --nginx -d eng-arzon.uz -d www.eng-arzon.uz`

## ⚠️ ВАЖНО: rate-limiting требует Nginx-прокси

`src/lib/rateLimit.ts` определяет IP посетителя из заголовков
`X-Forwarded-For`/`X-Real-IP`. Self-hosted Next.js (в отличие от Vercel) не
имеет встроенного доступа к IP клиента без прокси-сервера перед собой — пока
сайт открыт напрямую по `http://IP:3001`, **все посетители считаются одним
и тем же клиентом** для целей rate-limiting (защита всё равно работает,
но не различает разных людей). Как только настроен Nginx-прокси с двумя
заголовками выше — лимиты начинают работать корректно для каждого
посетителя отдельно, без изменений в коде.

## Флоу проверки объявлений (анти-фрод)

Чтобы исключить фейковые объявления, продавец не может опубликовать авто
напрямую — только подать заявку через `/sell`:

1. Продавец заполняет форму: данные авто, желаемая цена, был ли в аварии
   (с описанием повреждений, если да), фото техпаспорта с обеих сторон
   (перед/зад), 4 фото автомобиля по фиксированным ракурсам
   (перед, зад, левый бок, правый бок). VIN отдельно не запрашивается —
   он виден на фото техпаспорта.
2. Заявка сохраняется со статусом **«На проверке»** — недоступна на сайте.
3. Менеджер открывает `/admin/moderation`, смотрит фото техпаспорта
   (включая VIN), смотрит фото авто, видит, указал ли продавец
   аварийность, вручную сверяет цену продавца с объявлениями на других
   досках (OLX, Avto.uz) и вводит референсную цену в поле сверки.
4. **Одобрить** → авто публикуется (статус «В продаже»), продавцу приходит
   уведомление в Telegram (если привязан telegramId).
   **Отклонить** → авто получает статус «Отклонено» с указанной причиной,
   продавцу также приходит уведомление.

Эта проверка применяется только к новым заявкам с публичной формы.
Авто, добавленные напрямую через `/admin/cars/new`, публикуются сразу
(админ считается уже доверенным источником).

## Авто-снятие объявления по лимиту обращений

Чтобы на сайте не копились "мёртвые" объявления (продавец перестал
отвечать, цена нерыночная и т.п.), при каждом нажатии "Bog'lanish"
сайт фиксирует обращение через `/api/cars/:id/contact`.

- Считаются только **уникальные** посетители — связка IP-адреса и
  анонимного cookie (`arzonroq_vid`). Повторные нажатия одного и того
  же человека не увеличивают счётчик.
- При достижении **50 уникальных обращений** авто автоматически
  переводится в статус «Снято с продажи» (DELISTED).
- Продавец получает уведомление в Telegram постфактум (без возможности
  отложить или оспорить): *«Объявление снято — достигнут лимит 50
  обращений, опубликуйте заново, если автомобиль ещё актуален»*.
- Счётчик хранится в поле `Car.contactCount`, индивидуальные клики —
  в таблице `ContactClick` (на случай, если лимит нужно будет менять
  или анализировать историю обращений).

Это единственный автоматический механизм снятия объявлений — проверки
актуальности по таймеру (например, "висит 24ч без ответа продавца") нет.
Объявление остаётся опубликованным сколько угодно, пока не достигнет
лимита обращений или не будет изменено вручную в админке/через Telegram.

## ⚠️ ВАЖНО: вернуть Secure-cookie после настройки HTTPS

В `src/app/api/admin/login/route.ts` временно стоит `secure: false` для
cookie сессии админки — это позволяет тестировать логин по голому HTTP
(`http://IP:порт`) без домена и SSL. **Как только будет настроен домен и
HTTPS (Nginx + Let's Encrypt), обязательно вернуть `secure: true`**
(или `process.env.NODE_ENV === "production"`), иначе сессия админа будет
передаваться по сети в открытом виде — это угроза перехвата cookie.

## Известные ограничения MVP (что добавить дальше)

- Поиск по тексту (марка/модель) в фильтрах — точное совпадение; можно добавить
  autocomplete на основе реально существующих в базе значений.
- Привязка Telegram ID через /start доверяет контакту, который присылает
  Telegram (продавец делится номером через нативную кнопку). Это удобно, но
  теоретически через "Поделиться контактом" можно отправить чужой номер из
  адресной книги. Для продакшена можно добавить подтверждение SMS-кодом или
  ограничить привязку до момента, когда админ сам сгенерирует одноразовый код
  для конкретного продавца.
- Подсчёт уникальных обращений (IP + cookie) — это эвристика, не идеальная
  идентификация. Один человек с разных устройств/без cookie (режим инкогнито,
  очистка данных браузера) может засчитаться несколько раз; несколько разных
  людей в одной офисной/домашней сети с одним IP могут засчитаться как один.
  Для большей точности в будущем можно перейти на привязку к Telegram ID
  покупателя (если потребуется логин для звонка), но это усложнит UX.
